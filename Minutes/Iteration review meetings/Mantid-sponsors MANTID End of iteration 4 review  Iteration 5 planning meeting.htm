Team Discussions and Decisions

Visualization for 2D and 1D plots

We discussed the possibilities of tools for use visualising our data. The major points raised were:

1.       The solution needs to compare favourably with the tools that are in current use. One of the reasons that people do not like OpenGenie is down to the plotting.

2.       Users do not like GnuPlot.

3.       Whatever we select needs to be able to handle a large data volume (at least a 2D plot of 3200*2000).

The following were raised as possibilities for investigation.

1.       QTIplot Good 1D plotting, 2D plotting but not as good as matlab. Need to asses how well it handles the data volume.

2.       SciLab integration This depends on the ability to interface to this through C++. Early web searches have not proved fruitful.

3.       MatLab plotting through a precompiled MatLab application. An Inspection of the Matlab License suggests that the non competition clause of the deployment addendum of the Mathworks software license agreement appears to prevent this, and it would also prevent us using the GPL.

4.       Paraview a QT based application for visualising large data sets based on VTK. This would require a lot of development as it knows nothing about graph types. It also appears to rely on back end server farms to provide rendering performance.

Workspace data storage rework

This potential redesign was discussed raising the following points:

  1. The need for Poisson error models is important to LOQ, but not for some other instruments.
  2. There was a concern that the inheritance in this structure could cause performance issues.
  3. The grouping of DetecorNo with Yvalue prevented the multiple referenceing of homogenous time bins.
  4. Benchmark timings of the potential results will need to be performed before choosing a solution.
